### 2.2 Observer / ingestion (`src/observer/`)

Phase 0:
- `_close_position` (position_tracker) and `_process_trade` (trade_observer) wrapped in `async with conn.transaction():` so a mid-chain crash never leaves torn state.

Phase 1 (HP-1 ‚Äî the big one):
- `TRADE_OBSERVER_POLL_INTERVAL_S`: 30 ‚Üí 5
- ETag / `If-None-Match` round-trip on the data-api poll
- Bounded `asyncio.Queue(10_000)` between `_process_trade` (producer) and a new `_db_writer_loop` (consumer)
- Batches up to 200 rows / 100 ms committed as one tx; per-row `ON CONFLICT DO NOTHING` fallback on `UniqueViolationError`
- Backpressure: `wait_for(queue.put, 1s)` ‚Üí drop with counter increment

Phase 2:
- `PositionTracker` state persisted to `position_tracker_state` (migration 015); `warm_start(conn)` loads on engine boot BEFORE `tracker.start()`. DELETE chained into the Phase 0 close-tx. Eviction at 10 k LRU-by-open_time.

Phase 3 R1 ‚Äî data continuity backbone:
- **Continuous-cursor REST polling** ‚Äî Redis-stored `last_seen_id` per source/wallet replaces time-window queries. Cursor advances ONLY after batch commit; crash-replay handled by the trades_observed UNIQUE INDEX.
- **WS freshness watchdog** ‚Äî per-channel `last_msg` in Redis; >60 s silence triggers `force_reconnect` with bounded backfill (‚â§24 h).

Phase 3 R2 (pending):
- **Order-book imbalance pipeline** ‚Äî `OrderBookObserver` rolls up `book_quality_snapshots` into `orderbook_features_minute` (per-minute depth_imbalance, spread bps, microprice deviation).

### 2.3 Registry / Falcon client (`src/registry/`)

Phase 0:
- `markets.liquidity_score` source corrected: Falcon agent **574 ‚Üí 575** (Market Insights). The previously dead agent 575 now has a callsite.

Phase 1:
- `FalconClient` semaphore 1 ‚Üí 8 (env: `FALCON_MAX_CONCURRENCY`)
- `_backfill_wallet_trades` parallelized via `asyncio.gather(return_exceptions=True)` with `Semaphore(REGISTRY_BACKFILL_CONCURRENCY=20)`. Registry cycle 10-20 min ‚Üí ~5 min.

Phase 3 R1:
- **Event-driven Falcon refresh**: `LeaderRegistry.refresh_wallet(wallet, reason=‚Ä¶)` replaces the 30-min `FALCON_REFRESH_INTERVAL_S` as the primary path (timer retained as floor). Triggered by `event_bridge` consuming `trades:observed` for unknown wallets, by Telegram `/refresh`, or by the watchdog.
- **Gating**: per-wallet cooldown, asyncio.Event coalescing of concurrent refreshes, daily Falcon budget counter in Redis (`falcon:budget:YYYYMMDD` TTL 25 h, default 500/day).
- **`FalconKeyPool`** ‚Äî multi-key infra: env `FALCON_API_KEYS=k1,k2,k3`. Backward-compat with single `FALCON_API_KEY`. Per-key token bucket (60 RPM cap). Halves on HTTP 429 for 60 s, restores after.
- **In-flight call coalescing** ‚Äî identical concurrent calls share one HTTP request; 30 s post-completion TTL.
- **Conditional GET** (ETag / `If-Modified-Since`) on the 4 Falcon agents that support it; 304s don't count against the rate limit.

## 8. Single sentence per phase

- **Phase 0**: every production-safety bug the audit found was fixed ‚Äî atomic writes, the killswitch leak, wrong-Falcon-agent for liquidity, the gamma fee misinterpretation, and retention infra for 9 unbounded tables.
- **Phase 1**: data freshness ceiling collapsed from ~16 s to ~2-3 s end-to-end via 30 s‚Üí5 s polling, ETag conditional GET, micro-batched DB writes with bounded queue and backpressure, parallelized Falcon backfill, and a 12-metric Prometheus baseline.
- **Phase 2**: schema scales to 10 √ó  current data volume via `trades_observed` partitioning with `DROP PARTITION` retention, missing partial indexes and enum CHECKs land, `PositionTracker` survives restart via warm-start, and every Redis subscriber gets a dedicated client with auto-resubscribe on reconnect.
- **Phase 3 Round 1**: every structural cause of data-acquisition pauses removed ‚Äî continuous cursor (no window gaps), event-driven Falcon refresh (no 30-min timer pause), WS freshness watchdog (no silent stalls), multi-key Falcon throughput, in-flight call coalescing, Redis Streams CDC with durable replay, ingest-health watchdog with auto-recovery, and `IngestSourceDown if silent >30min` Prometheus alert.
- **Phase 3 Round 2** (in flight): the model-validity work ‚Äî bivariate Hawkes turns `Œ±/Œº` into a real causal measure, point-in-time feature store closes the train/serve skew on `liquidity_score`, and the order-book imbalance pipeline adds the audit's highest-ROI new feature trio.
